# 機能定義書：タイムライン

## 概要

タイムラインには2種類ある。全ユーザーの投稿を新着順で表示する**全体タイムライン**と、フォロー中のユーザーの投稿のみ表示する**フォロータイムライン**。タブUIで切り替えられる。

---

## 機能一覧

| # | 機能 | 説明 |
|---|------|------|
| 1 | 全体タイムライン表示 | 全ユーザーの投稿を新着順で一覧表示する |
| 2 | フォロータイムライン表示 | フォロー中のユーザーの投稿のみ新着順で表示する |
| 3 | タイムライン切り替え | タブUIで全体／フォローを切り替えられる |

---

## 詳細仕様

### タイムラインの種類

| 種類 | 表示対象 | APIパラメータ | デフォルト |
|------|---------|-------------|-----------|
| 全体タイムライン | 全ユーザーの投稿 | `GET /api/v1/posts?type=global` | ✓ |
| フォロータイムライン | フォロー中ユーザーの投稿のみ | `GET /api/v1/posts?type=following` | |

- どちらも `created_at` の降順（新着順）で表示
- フォロータイムラインでフォロー中ユーザーが0人の場合は「まだ投稿がありません。ユーザーをフォローしてみましょう。」を表示

---

### 各投稿カードに表示する情報

| 要素 | 説明 |
|------|------|
| 投稿者アイコン・ユーザー名 | クリックでプロフィール画面へ遷移 |
| 投稿日時 | |
| 投稿本文 | 最大280文字 |
| 添付画像サムネイル | 最大4枚 |
| いいねボタン（トグル） | カード上でいいね可能。いいね済みは色変更 |
| いいね数 | |
| コメント数 | クリックで投稿詳細画面へ遷移 |

---

## APIエンドポイント

| メソッド | エンドポイント | 認証 | 説明 |
|---------|--------------|------|------|
| GET | `/api/v1/posts?type=global` | 必要 | 全体タイムライン（全ユーザーの投稿・新着順） |
| GET | `/api/v1/posts?type=following` | 必要 | フォロータイムライン（フォロー中のみ・新着順） |

---

## 実装済み仕様（v1）

> ※ 以下は現在実装済みの仕様です。上記の仕様と差異がある場合は後続対応で統合予定。

### ページング

| パラメータ | 型 | デフォルト | 説明 |
|-----------|-----|---------|------|
| `page` | integer | 0 | ページ番号（0始まり） |
| `size` | integer | 20 | 1ページあたりの取得件数 |

レスポンス形式:

```json
{
  "posts": [...],
  "hasNext": true,
  "totalCount": 42
}
```

### 無限スクロール

- フロントエンドは `IntersectionObserver` でリスト末尾の sentinel 要素を監視
- sentinel が表示領域に入ったタイミングで次ページを自動取得（`page + 1`）
- `hasNext: false` になったら追加取得を停止

### 新着バナー（ポーリング）

- ログイン後、60秒間隔で `GET /api/v1/posts?after={latestPostId}` をポーリング
- WebSocketは使用しない
- 新着件数が1件以上の場合、タイムライン上部に「{n}件の新しい投稿があります」バナーを表示
- バナークリック時にタイムラインを先頭から再取得・バナーを非表示

### 今回のスコープ外（後続対応）

- フォロータイムライン（`type=following`）
- タブUIによる全体／フォロー切り替え
