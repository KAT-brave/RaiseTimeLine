# 要件定義書

## 1. アプリ概要

| 項目 | 内容 |
|------|------|
| アプリ名 | RaiseTimeLine |
| 目的 | 学習目的のSNS風Webアプリ |
| 対象ユーザー | 受講生・個人（複数ユーザーの同時利用を想定） |
| フロントエンド | React（TypeScript） |
| バックエンド | Spring Boot（Java） |
| データベース | PostgreSQL |
| 画像ストレージ | AWS S3 |
| 認証方式 | メールアドレス + パスワード（JWT） |

---

## 2. 機能一覧

機能の詳細は [features.md](features.md) および `features/` 配下の各機能定義書を参照してください。

| # | 機能 | 概要 |
|---|------|------|
| 1 | ユーザー登録・ログイン | メール+パスワードによる登録・ログイン・JWT認証 |
| 2 | タイムライン | 全体タイムライン・フォロータイムラインの表示・タブ切り替え |
| 3 | 投稿 | テキスト（最大280文字）＋画像（最大4枚）の投稿作成・編集・削除 |
| 4 | コメント | 投稿へのコメント投稿・削除・コメント数表示 |
| 5 | いいね | 投稿へのいいね追加・取り消し・いいね数表示 |
| 6 | プロフィール・フォロー | プロフィール表示・編集・フォロー/フォロワー管理 |
| 7 | ユーザー検索 | ユーザー名の部分一致検索でユーザーを探してフォロー |

---

## 3. 非機能要件・制約

| 項目 | 内容 |
|------|------|
| 投稿文字数 | 最大280文字 |
| 画像枚数上限 | 1投稿につき最大4枚（JPEG/PNG/GIF） |
| 画像保存先 | AWS S3（presigned URL方式でアップロード） |
| インプレッション数 | 表示しない |
| リツイート機能 | 提供しない |
| OAuth認証 | 提供しない（メール+パスワードのみ） |

---

## 4. ユーザーストーリー

詳細は [features.md](features.md) を参照してください。

- ユーザーとして、サインアップ画面からアカウントを作成してログインしたい
- ユーザーとして、全ユーザーの投稿を全体タイムラインで見たい
- ユーザーとして、フォロー中のユーザーの投稿だけをフォロータイムラインで見たい
- ユーザーとして、テキスト（最大280文字）と画像を含む投稿をしたい
- ユーザーとして、自分の投稿を編集したい
- ユーザーとして、気に入った投稿にいいねしたい
- ユーザーとして、投稿にコメントして交流したい
- ユーザーとして、ユーザー名で他のユーザーを検索してフォローしたい
- ユーザーとして、投稿・コメントのユーザー名をクリックしてプロフィールを確認しフォローしたい
- ユーザーとして、自分のフォロー・フォロワー一覧を確認したい
